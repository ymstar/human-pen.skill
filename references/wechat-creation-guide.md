# 微信公众号创作规范手册

## 一、核心理念：零 AI 痕迹 + 公众号特有调性

微信公众号使用腾讯朱雀 AI 检测系统（7 层漏斗检测），AI 痕迹过高会被限流、下架。

目标是让文章读起来像一个真实的人在发朋友圈、聊微信，而不是在读论文或看机器生成的报告。

**与知乎的核心差异：**
- 公众号更口语、更碎片、更注重点击
- 公众号算法推荐权重更大（公域 70% vs 私域 30%）
- 公众号段落要更短、排版要更"轻"
- 公众号情感表达要更自然、更有层次

---

## 二、腾讯朱雀 AI 检测：7 层漏斗机制

公众号使用朱雀系统对内容进行 7 层检测，任意一层触发都会限流：

| 层级 | 检测维度 | 违规表现 | 后果 |
|------|----------|----------|------|
| L1 | 语言模式 | 句子长度统一、词汇多样性低 | AI痕迹过高降权 |
| L2 | 句式结构 | 固定模板（首先/其次/最后） | 模板化严重限流 |
| L3 | 逻辑特征 | 论证过于工整、缺乏跳跃性 | 机械感强不推荐 |
| L4 | 语义深度 | 概念堆砌、缺乏具体案例 | 空泛内容过滤 |
| L5 | 情感颗粒度 | 情绪表达平淡、缺乏层次 | 情感扁平降权 |
| L6 | 原创度 | 与已有内容相似度高 | 同质化不推荐 |
| L7 | 人机协同 | 综合判断人工参与度 | 纯AI直接限流 |

**安全线：AI 浓度控制在 20% 以下**

检测工具：https://matrix.tencent.com/ai-detect/

## 三、去除 AI 痕迹：24 种模式检查清单

### 📝 内容模式（6种）——必须改

1. **禁止宏大叙事** → 删除"这不仅是一个XX，更是XX的革命"之类的话
2. **禁止模糊引用** → 不用"行业专家认为""据悉""有报道称"，要不就别说，要说就给具体来源
3. **禁止泛泛描述** → 少用"正在改变""不断演变"，要给出具体结论
4. **禁止广告语** → 不用"无缝""直观""强大""一站式"，用具体功能替代
5. **禁止模糊归因** → 不用"研究表明"，要么给具体论文/报告，要么说"我自己的经验是"
6. **禁止提纲式结尾** → 不要写"挑战与展望"，要写就写具体的有观点的总结

### 🔤 语言和语法模式（6种）——必须改

1. **AI 高频词黑名单** → 严禁：此外、至关重要、深入探讨、强调、格局、无缝、直观、展示、证明、充满活力、值得注意的是、事实上、总而言之
2. **禁止同义词循环** → 同一个概念不要换不同词反复说，直接用同一个词
3. **禁止三段式法则** → 不要总把事物分成三组，两组、四组、五组都行，或者不分组
4. **禁止否定式排比** → 不用"不仅仅是……更是……"，不用反复"不是A而是B"
5. **允许用"是"** → 不要刻意回避系动词，该用就用
6. **禁止虚假范围** → 不用"从A到B"式的空洞描述，除非真有意义

### 🎨 风格模式（6种）——必须改

1. **破折号少用** → 用逗号、句号、括号替代
2. **粗体不过度** → 只在真正关键的地方加粗，不要每段都加
3. **列表少用** → 不要每个小节都是列表，要有连贯的自然段落
4. **emoji 可用** → 公众号可以适当用表情，但别过度
5. **引号用直引号** → 不要用弯引号 ""
6. **标题平实** → 不要过度装饰，但可以有情绪

### 💬 交流模式（6种）——必须改

1. **禁止客套话** → 删除"希望对您有帮助""如有疑问欢迎留言"
2. **禁止截止日期声明** → 删除"截至XX年XX月"
3. **禁止谄媚语气** → 删除"很荣幸""感谢您的阅读"
4. **禁止填充短语** → 删除"事实上""实际上""当然""总而言之"
5. **不要过度限定** → 不要到处加"可能""也许""在一定程度上"
6. **禁止空洞结论** → 不用"相信未来会更好""让我们拭目以待"

## 四、公众号口语化技巧：让你的文章"像人说话"

### 🗣️ 口语化词语替换表

公众号要求更自然的聊天感，以下是常用替换：

| AI 味表达 | 口语化替换 |
|-----------|-----------|
| 然而 | 但 |
| 因此 | 所以、于是 |
| 观看 | 看 |
| 进行处理 | 搞一下、处理下 |
| 非常重要 | 特关键、老重要了 |
| 综上所述 | 说白了、这么回事 |
| 众所周知 | 说起来 |
| 实际上 | 其实呢 |
| 可以发现 | 你有没有发现 |
| 接下来 | 那么 |
| 首先 | 先说说、第一件事 |
| 其次 | 然后、再一个 |
| 最后 | 搞完了、还有一点 |

### 😤 语气词和叠词要用起来

公众号是"聊天"不是"写报告"，适当用语气词：

- 叠词："试试看""慢慢来""好好说""细细想"
- 语气词："呀""呢""啊""嘛""呗"
- 反问句："你真的懂吗？""这不是明摆着吗？"
- 感叹句："绝了！""太牛了！""没想到吧？"

### ❓❗… 标点符号的情绪功能

| 标点 | 情绪效果 | 适用场景 |
|------|----------|----------|
| ！ | 兴奋、惊讶、强调 | "太绝了！" |
| ？ | 疑问、反问、引导思考 | "你真的觉得这样行？" |
| …… | 意味深长、留白 | "事情没那么简单……" |
| " " | 讽刺、反讽 | "我可真是"聪明"" |
| —— | 转折、补充 | 少用，可用"但是"代替 |

### 📏 句子要短，段落要碎

**对比示范：**

❌ AI味长句：
"在当今数字化转型的浪潮下，企业对于高效的协作工具的需求变得越来越迫切，而选择一款适合的工具则显得尤为重要。"

✅ 口语化短句：
"企业想选个好用的协作工具？挺难的。市面上选择太多，挑花眼是常事。"

**段落规范：**
- 每段不超过 3 行（手机屏幕阅读）
- 段与段之间留白
- 重点内容单独成段

## 五、公众号内容结构规范

### 📏 字数标准

| 类型 | 字数范围 | 特点 |
|------|----------|------|
| 干货类 | 1200-2000字 | 快、准、狠，直接给方法 |
| 故事类 | 1500-2500字 | 有情节、有情感、有共鸣 |
| 资讯类 | 800-1200字 | 快讯、简短点评 |

### 🏃 开头决定生死（前3段）

公众号读者滑动速度快，前3段没吸引住就划走了。

**好的开头：**
1. 直接抛出痛点或反直觉观点
2. 用一个问题或故事开头
3. 3行内亮出核心价值

**示例：**
> 你还在用 Excel 管项目？难怪天天加班到半夜。
> 
> 我见过太多团队，项目管理全靠"表哥""表姐"手动更新。
> 
> 今天聊点不一样的——怎么用看板工具真正管好项目。

**坏的开头：**
- ❌ "在当今数字化转型的背景下……"
- ❌ "随着技术的不断发展……"
- ❌ "项目管理是企业管理的重要组成部分……"

### 📑 主体结构：短段 + 小标题

公众号小标题要：
- 简短有力（5-10字）
- 有情绪或悬念
- 不要太正式

**示例小标题风格：**
- "踩坑实录"
- "血的教训"
- "没想到吧？"
- "说个真事"
- "这个坑别踩"

### 🎯 结尾：给明确判断或行动

- 不要写空洞的"展望"
- 要么给结论，要么给行动
- 可以留悬念引导互动

## 六、排版视觉规范

### 📐 基础排版

- 字号：正文 15-16px，标题 18-20px
- 行距：1.5-1.8 倍
- 页边距：两侧留白 10-15px
- 段距：段与段之间空一行

### 🖼️ 配图要求

- 每 500-800 字配一张图
- 图片风格：轻松、真实、可以有表情包
- 封面图要醒目、有情绪
- 可以用截图、实拍图，不要全是设计感强的图

### ✨ 重点突出技巧

- 加粗：关键词加粗，不要整句加粗
- 变色：重点内容用不同颜色（但别太多颜色）
- 引用块：名人名言、金句用引用样式
- 数字突出：大数字可以用特殊样式

|------|----------|------|

## 八、推荐算法与流量池机制

### 📊 核心指标权重（2026年）

| 指标 | 权重 | 优化重点 |
|------|------|----------|
| 完读率 | 30% | 开头抓人、结构清晰 |
| 点击率 | 25% | 标题、封面 |
| 互动率 | 25% | 内容引发共鸣 |
| 关注转化 | 15% | 持续提供价值 |
| 账号权重 | 5% | 日常养号 |

### 🏊 流量池晋级标准

1. **初始池（100-500人）**：点击率 ≥ 8% 进入
2. **二级池（1000-3000人）**：完读率 ≥ 40%，互动率 ≥ 3%
3. **三级池（1万-10万人）**：综合数据优秀
4. **爆款池（10万+）**：数据爆发

### 🎯 重点：点击率和完读率

- **点击率靠标题和封面**：要会起标题，但不能做"标题党"
- **完读率靠开头和结构**：前3段必须抓住人，中段不能拖沓

## 九、2025年微信AI内容新规

### ⚠️ 强制要求

1. **AI生成内容必须标注**
   - 未标注"AI生成"的文章会被限流/下架
   - 建议在文末或开头注明

2. **禁止非真人自动化创作**
   - AI生成 + 改写 + 拼接 + 搬运 → 违规
   - 脚本批量发布 → 违规

3. **AI辅助创作可以，但必须人工审核**
   - AI可以辅助，但必须人工校对、修改
   - 最终内容要体现人工参与痕迹

### ✅ 合规创作建议

- AI浓度控制在 20% 以下（用朱雀工具检测）
- 保持内容有个人的独特视角和经验
- 融入真实案例、情感、个人观点

## 十一、账号运营节奏

### 📅 输出频率
- 建议每天 1 篇，或每周 4-5 篇
- 保持规律发布，培养读者习惯

### 🎯 内容来源
- 追热点写新文
- 系列教程/专题
- 读者问题/留言选题
- 个人经验/踩坑分享

### 📈 领域定位
- 保持垂直度
- 公众号比知乎更强调"人设"
- 要有自己的风格和态度

## 九、4 层改写方法论（来自 ai-article-humanizer 技能）

核心原则：不是换词，是换思维。AI 写作 = 全面/平衡/不犯错，人类写作 = 偏/不全面/有情绪/有立场。

### 第一层：结构层改写
- 打破模板结构（总分总、并列罗列），只保留一个点深入讲

### 第二层：例子层改写
- 补充"不完美细节"：犹豫、失败、吐槽、转折

### 第三层：语气层改写
- 把"太确定"换成"留有余地"：可能、我觉得、也不一定

### 第四层：删减层改写
- "正确但没信息量"的句子直接删掉

### 真人真话测试
「这段话像是给所有人看的正确答案，还是一个真人在说自己的真话？」

### 追加禁令
- 禁止成语：因噎废食、千人一面、与时俱进、不可或缺、相得益彰等
- 禁止书面词：旨在、致力于、亟待、引人注目、深刻变革等
- 禁止套话：远超想象、协同发展、共同创造、美好未来等
- 禁止句式："唯有...才能...""不仅...更...""随着...的发展..."
